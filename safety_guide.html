핵심 원칙
① 모든 입력 페이지는 모바일 우선 + 단순한 폼
② 데이터는 Supabase로 실시간 공유 (브라우저 다 켜놓으면 자동 갱신)

2 패트롤 (환경운영점검) 운영법

담당자: 박제성 부장 (본사) · 임문호 이사 (화성공장) — 패트롤 시 모바일 입력

운영 흐름

STEP 1
QR 스캔 또는 링크 접속
현장에서 사이트(수원/화성) 선택
STEP 2
구역별 체크
각 구역 OK/NG 2버튼 + 사진 첨부 (NG는 자동으로 지적사항 등록)
STEP 3
서명 + 저장
점검자 서명 → Supabase 자동 동기화
STEP 4
대시보드 자동 반영
관리자 대시보드 [패트롤+TBM 통합]에서 즉시 확인

주기

법규상 월 2회 이상 권장. 안전관리자(대한산업안전협회) 방문일도 월 2회.

인쇄용 양식

대시보드의 [📨 공문서 관리 대장] 또는 직접 patrol_print.html에서 A4 양식 출력 가능.

3 TBM (Tool Box Meeting) 운영법

담당자: 관리감독자 3명 (사출/도장/조립) — 매일 작업 전 5분 회의 후 입력

팀별 링크 (URL 파라미터로 분기)

사출 주간조
김민수 부장
tbm.html?team=injection_day
사출 야간조
김민수 부장 위임
tbm.html?team=injection_night
도장
양춘향 과장
tbm.html?team=paint
조립
임문호 이사
tbm.html?team=assembly

운영 흐름

  1. 작업 전 5분 TBM 회의 — 당일 위험 포인트, 안전 수칙, 신규 작업자 배치 등 공유
  2. tbm.html에 입력 — 일자, 참석자, 작업 내용, 위험 요인, 안전 조치, 보호구 점검
  3. 저장 → 자동 동기화
  4. 관리자가 캘린더에서 확인 — 대시보드 [패트롤+TBM 통합] → TBM 캘린더에서 일자별 클릭
  5. 인쇄 양식 — 캘린더에서 일자 클릭 → A4 인쇄 (작업일지로 보관)

주기

매 작업일 1회 (휴무일 제외). 미입력 시 다음날 대시보드에서 누락 표시됨.

3.5 관리감독자 일일점검 운영법

담당자: 김민수 부장(사출) · 양춘향 과장(도장) · 임문호 이사(조립) — 매일 25개 항목 O/X 점검

왜 별도 페이지인가

관리감독자는 본인 영역만 매일 점검하면 됨. 33개 업무·조직도·알림 등이 섞인 안전보건관리자 페이지(safety_manager.html)는 너무 무거움. 그래서 패트롤·TBM처럼 본인 점검만 할 수 있는 전용 페이지(supervisor_check.html)를 별도로 두었습니다.

운영 흐름

STEP 1
QR 스캔 / 링크 접속
본인 링크(?leader=kim/yang/lim) 또는 첫 화면에서 본인 카드 클릭. 한 번 선택하면 기기에 저장
STEP 2
5개 영역 × 5항목 = 25개 O/X
상단 진행률 바 + 영역별 자동 접힘 (모두 O면 자동 정리)
STEP 3
특이사항 입력 + 저장
미이행(X) 사유, 조치 내용 자유 기재 → [저장] 버튼
STEP 4
최근 14일 이력 확인
하단 표에서 본인 입력 이력. 행 클릭 시 해당 날짜로 점프하여 수정 가능
STEP 5
관리자가 월별 매트릭스로 통합 확인
박재걸/박제성이 safety_manager.html [월별 누적 매트릭스]에서 5×31 그리드로 한달치 점검 (실시간 동기화)

주기

매 작업일 1회. 휴무·휴가 시 입력 불요. 14일간 누락되면 누적 매트릭스에 빈 칸으로 표시.

데이터

저장 키: safetyMgrDaily
형식: { '2026-04-28__김민수 부장': { items: {1:'O', 2:'X', ...}, notes: '...' }, ... }
같은 키를 safety_manager.html의 월별 매트릭스가 읽어와서 통합 표시. 따로 동기화 작업 불필요.

QR 분리 운영: 각 감독자 휴대폰 카메라로 본인 QR만 스캔하도록 게시. 다른 감독자 데이터는 안 보이고, 본인 14일 이력만 표시됩니다. 박재걸/박제성(관리자)는 safety_manager.html에서 셋 다 통합 조회.

4 안전보건관리자 페이지 운영법

담당자: 박재걸 전무 + 박제성 부장 — 33개 안전보건 업무 + 일일 체크리스트 관리

이 페이지에서 하는 일

기능 1
이행 현황 요약
전체 33건 중 지연/임박/정상/이벤트형 한눈에
기능 2
알림 패널
마감 7일 이내 + 마감 지난 업무. 이메일 발송 버튼
기능 3
조직도 트리
이름 클릭 → 옆 패널에 그 인물 담당 업무 목록
기능 4
33개 업무 카드
필터(안전/도급/보건/지연/임박) + 카드별 [완료 기록] 버튼
기능 5
관리감독자 일일 체크
25개 항목 O/X, 날짜·감독자별 저장
기능 6
월별 누적 매트릭스
5분류 × 일자 그리드. 한 달 이행 한눈에

일일 운영 흐름

  1. 아침 — 페이지 열기 → 알림 패널에서 오늘 마감 업무 확인
  2. 업무 완료 시 — 해당 업무 카드 [완료 기록] 클릭 → 다음 마감일 자동 계산
  3. 관리감독자 입력 — 김민수/양춘향/임문호가 각자 [관리감독자 일일 체크리스트]에서 25개 O/X 입력 → 저장
  4. 월말 — [월별 누적 매트릭스]에서 한달치 점검
  5. 매일 오전 9시 — GAS가 자동으로 마감 임박/지연 업무를 이메일 발송 (설정 후)

6 이메일 알림 설정 (Google Apps Script)

매일 오전 마감 임박/지연 업무를 자동 이메일로 받기 — 1회 설정으로 끝

왜 GAS인가?

HTML 페이지(GitHub Pages)는 정적이라 서버에서 메일을 보낼 수 없어. Google Apps Script가:

  • 무료 (일 100건 발송 한도, 충분)
  • 구글 계정 그대로 사용 — 별도 SMTP/API 키 불필요
  • 시간 기반 트리거로 매일 자동 실행 가능
  • Supabase에서 직접 데이터 조회

설정 절차 (1회만)

  1. GAS 프로젝트 생성 — script.google.com 접속 → [새 프로젝트]
  2. 코드 붙여넣기 — 프로젝트 폴더의 safety_notify.gs 전체 내용 복사 → Code.gs에 붙여넣기
  3. 저장 — 프로젝트명: SSAT_Safety_Notify
  4. 웹 앱 배포 (수동 발송용)
    • [배포] → [새 배포] → 유형: 웹 앱
    • 실행: 본인 (ejkim2933@gmail.com)
    • 액세스: 모든 사용자
    • 배포 → URL 복사
    • safety_manager.html에서 [이메일 발송] 첫 클릭 시 URL 입력 → 자동 저장
  5. 자동 다이제스트 트리거 (매일 자동)
    • 왼쪽 시계 아이콘 [트리거] → [트리거 추가]
    • 실행할 함수: sendDailyDigest
    • 이벤트 소스: 시간 기반
    • 유형: 일별 타이머 / 시간: 오전 8-9시
    • 저장 → 첫 실행 시 Gmail 발송 권한 동의
완료: 매일 오전 8-9시 사이 자동으로 마감 임박/지연 업무 이메일이 ejkim2933@gmail.com에 도착합니다.
수신자 변경: safety_notify.gs의 DEFAULT_RECIPIENTS 수정 후 GAS 재저장.

수동 발송도 함께

safety_manager.html의 알림 카드 [이메일 발송] 버튼을 클릭하면 즉시 이메일이 갑니다. 출장·휴가 전 사전 점검할 때 유용.

8 자주 묻는 질문

Q. 인터넷 안 될 때도 입력할 수 있나요?

네, 모든 입력 페이지는 localStorage(브라우저 저장소)에 먼저 저장한 뒤 Supabase에 동기화합니다. 오프라인 입력 후 인터넷 복구 시 자동 업로드.

Q. 한 사람이 여러 페이지 동시에 켜놔도 되나요?

네, Supabase 실시간 채널이 모든 페이지를 자동 동기화. 한쪽에서 수정하면 다른 페이지도 즉시 반영.

Q. 데이터를 잘못 저장했어요. 복구 가능한가요?

각 페이지에 [지우기]/[삭제] 버튼이 있고, Supabase에는 최근값만 저장됩니다. 과거 시점 복구는 어렵습니다. 중요 자료는 월말에 인쇄하여 보관 권장.

Q. 비밀번호를 잊었어요.

대시보드 설정 페이지 진입 비밀번호: 김은진 (safety_dashboard.html의 SETTINGS_PASSWORD 상수, 변경 시 수정).

Q. 새로운 업무를 추가하고 싶어요.

다음 두 파일을 함께 수정해야 합니다:

  1. safety_manager_data.js의 SAFETY_TASKS 배열 끝에 새 객체 추가
  2. safety_notify.gs의 SAFETY_TASKS_GS 배열 끝에도 동일 항목 추가 (자동 이메일이 참조)

주기 코드는 위 표를 참고하세요. 수정 후 GAS는 [저장] 한 번만 누르면 됩니다.

Q. 관리감독자가 일일 체크리스트를 빼먹으면 어떻게 알 수 있나요?

현재는 [월별 누적 매트릭스]에서 빈 칸으로 표시됩니다. 자동 알림 메일에는 일일 체크 누락은 포함되지 않으니, 향후 필요시 GAS 스크립트에 누락 검사 로직을 추가할 수 있습니다.

Q. Supabase 무료 플랜이 정지되면?

3일마다 자동 ping(GitHub Actions의 supabase-keepalive.yml)으로 정지를 방지하고 있습니다. 만약 정지되면 Supabase 대시보드에서 [Resume] 클릭으로 즉시 복구.
