주재원 퇴직급여 비용 계상 처리 가이드

Руководство по учёту расходов на выходное пособие для командированных сотрудников
기준일 : 2026. 04. 20. 대상 인원 : 법인장 1명 + 주재원 부장 2명 (총 3명) 문서 번호 : SSAT-RU-2026-001
📌 처리 목적

주재원 3명의 퇴직급여 재원은 러시아 법인이 현지 별도 계좌로 이미 운용 중입니다.
이번 가이드는 이 금액이 러시아 법인 손익계산서(P&L)에 비용으로 정확히 반영되고 있는지 재점검하는 기회입니다.

① 원가 정확도
퇴직급여비용을 법인 운영비에 포함 → 실제 인건비 원가 반영
② 단가 협상 근거
러시아 현지 한국 대기업 납품 시 원가 기반 판매가 산정·네고 자료로 활용
③ 손익 투명성
법인 영업이익(Operating Profit) 산정 시 누락 비용 없이 정확한 수익성 파악
※ 현금 지출은 없습니다. 퇴직 시 러시아 법인이 운용 중인 현지 별도 계좌에서 지급됩니다.
※ 회계 처리 방식: Option A — 충당부채 계상 (현행 구조 유지). 별도 계좌 현금은 그대로 두고, 장부에만 비용·충당부채를 추가 계상합니다.
▶ OPTION A · 현행 적용 방식 충당부채 계상 (별도 계좌 현행 유지)
  • 러시아 현지 별도 계좌 현금은 그대로 유지
  • 충당부채를 법인 장부에 추가 계상
  • P&L에 퇴직급여비용 즉시 반영
  • 원가 계산에 포함 → 납품 단가 근거
  • РСБУ 기준 비용: 계상 시점 즉시 비용 인식
  • 현재 구조에서 즉시 실행 가능

월 계상액 산정 방법
계산식

1인당 월 퇴직급여 해당액

월 계상액  =  해당 월 급여 총액  ÷  12
※ 퇴직금 = 계속 근로 1년당 30일분 평균임금 (한국 근로자퇴직급여보장법 기준)
예시) 월 급여 700만원 → 월 계상액 = 700만원 ÷ 12 = 583,333원
3인 합산 월 계상 총액은 3인의 각 급여 기준으로 개별 산정 후 합산
2
매월 말 회계 처리 (분개)
처리 원칙 러시아 법인이 현지 별도 계좌로 재원을 직접 운용 중이므로, 대변은 법인 자체 충당부채(퇴직급여 충당금) 계정으로 처리합니다. 실제 지급은 퇴직 시 해당 러시아 현지 계좌에서 이루어집니다. 본사 간 자금 이동 없음.

환율 처리 기준
환율

원화(KRW) ↔ 루블(RUB) 환산

구분적용 방법
급여가 루블로 지급되는 경우 루블 기준 급여로 계상 → 법인 장부 루블 기준 유지
급여가 원화로 지급되는 경우 원화 기준 계상 → 당월 말 환율로 루블 환산 기재
환율 기준 러시아 중앙은행(ЦБ РФ) 공시 환율 적용 (법인 내부 규정 환율 있는 경우 우선 적용)
환율 주의 루블화 변동성이 크므로, 분기별 정산 시 환율 차이(환차손익)가 발생할 수 있습니다. 법인 내부 회계 규정에 따라 적용 환율 기준을 사전에 확정하고 일관되게 적용하시기 바랍니다.

원가 반영 및 납품 단가 협상 활용
핵심 활용 목적 — 러시아 현지 한국 대기업 납품가 산정 충당부채 계상으로 P&L에 비용이 잡히면, 이 금액을 법인의 실질 운영원가에 포함시켜 납품 단가 산정 및 바이어(한국 대기업) 네고 시 근거 자료로 활용합니다.

5
퇴직연금 정비 — 이참에 재점검
재점검 목적 퇴직급여 비용 계상을 시작하는 시점에 맞춰, 현재 주재원 3인의 퇴직급여 관련 사항 전반을 재점검합니다. 누락·오류 항목은 이번 기회에 정비합니다.
  • 근속 기산일 재확인 (3인 각각) → 파견 발령서 / 러시아 법인 근로계약서 기준으로 실제 근속 시작일 확인. 법정 퇴직금 계산의 출발점.
  • 법정 퇴직금 정확 계산 vs. 별도 적금 계좌 잔액 대조 → (근속연수 × 평균임금 30일분) 기준 법정 퇴직금 산정 후 현재 적금 잔액(만기 예상액 포함)과 비교. 부족분 발생 시 즉시 파악.
  • 적금 잔액 ↔ 충당부채 누계액 정기 대조 (분기별 권고) → 충당부채 계상을 시작한 이후, 매 분기마다 장부상 충당부채 누계액과 현지 적금 계좌 잔액을 대조 확인. 괴리 발생 시 원인 파악 후 조정.
  • 급여 기준 통화 및 평균임금 산정 방식 통일 → 루블 지급 / 원화 지급 / 혼합 지급 여부 확인. 퇴직금 계산 시 어느 통화 기준으로 산정할지 본사와 합의.
  • 충당부채 계상 시작 시점 결정 — 당월 vs. 소급 → 당월부터 시작: 손익 영향 분산 (권고). 과거분 소급 계상: 당기 비용 일시 반영 → 본사 사전 승인 필요.
  • 러시아 법인 계정과목 확정 → Прочие расходы 활용 또는 별도 계정 신설 여부 결정. 법인 회계 시스템 내 계정코드 부여.
  • 중간정산 가능 여부 검토 (필요 시) → 직원 요청 또는 회사 정책 변경 시 중간정산 가능. 근로자퇴직급여보장법상 사유 해당 여부 본사 노무 담당 확인.
  • DC형(확정기여형) 전환 — 적용 대상 및 조건 명확히 구분 → DC형은 한국 금융상품(한국 세법 기준)입니다. 한국 본사 소속 직원에게 적용하는 개념으로, 러시아 법인에서 납입한다고 해서 러시아 법인세 세제 혜택은 발생하지 않습니다. ⚠️ 러시아 법인 관점에서 DC형이 의미 있으려면:
    · 한국 본사에 과세 이익(흑자)이 있을 경우 한국 법인세 손금 처리 논의 가능 — 러시아 법인에 직접 세제 혜택 없음
    · 러시아 법인 자금을 한국 DC 계좌로 이전 시 러시아 외환통제(валютный контроль) 적용 — 별도 법무 검토 필요
    · 현재 구조(러시아 법인이 러시아에서 벌고 러시아에서 지출)에서는 충당부채 방식(Option A)이 현실적
